## Initial verdict

❌ **REJECT** — blocking seam found in apply_reingest.

### Blocking seam

`src/core/reconciler.rs` still let an existing page mutate through apply_reingest() when that page has zero total raw_imports history on the stat-diff apply path.

- Modified-path updates: existing_page_id = Some(page_id), mutate pages/file_state before rotation
- rotate_active_raw_import() treats row_count == 0 as acceptable bootstrap
- New-file paths can hit same seam when existing_page_id is None but load_existing_page_identity() resolves already-existing page by slug

**Violation:** Zero-active OR zero-total history must not slip through to mutation on changed/new-file paths.

### Acceptable controls (kept)

- ✅ full_hash_reconcile unchanged-hash narrow: self-heals file_state/last_full_hash_at only, no raw_imports rotation
- ✅ Full-hash preflight aborts on zero-active and zero-total before metadata self-heal
- ✅ UUID preservation explicit at render seam: render_page() re-emits persisted pages.uuid as gbrain_id
- ✅ Tasks defer restore/remap/UUID-writeback surfaces; no false completion claims

### Required narrowest repair route

1. Split apply-time raw-import behavior: **true create** vs **existing-page update** before mutating pages/file_state
2. For any path resolving to existing page (explicit existing_page_id OR slug match), require exactly one active historical row
3. Reject both zero-active and zero-total cases with InvariantViolationError
4. Add regressions:
   - stat-diff modified path with zero total raw_imports
   - stat-diff new-file/slug-match path with zero total raw_imports
   - rollback proof: pages and file_state unchanged on those aborts
