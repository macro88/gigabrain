## Problem

Nibbler's adversarial review identified a safety gap: the conservative hash-rename guard in `src/core/reconciler.rs` used whole-file size for the ≥64-byte threshold, allowing template notes with large frontmatter and tiny body to incorrectly satisfy the check and inherit the wrong page identity.

## Repair (Narrow)

Hardened the hash-rename guard to check post-frontmatter body size only:

1. **MissingPageIdentity.body_size_bytes** = `compiled_truth.trim().len() + timeline.trim().len()`
2. **NewTreeIdentity.body_size_bytes** = `body.trim().len()` where `body` is returned by `parse_frontmatter()`
3. **hash_refusal_reason()** gates on `body_size_bytes < 64`, not whole-file size
4. Refusal reason strings renamed for clarity: `missing_below_min_bytes` → `missing_below_min_body_bytes`
