## Context

Nibbler rejected Batch G on adversarial seam: apply_reingest() on stat-diff path allowed existing page with zero-total raw_imports rows to proceed through rotate_active_raw_import(), which treats row_count == 0 as bootstrap state. This silently created first history row for existing page instead of failing closed.

Two paths vulnerable:
1. **Modified-path:** ApplyAction::Reingest { existing_page_id: Some(id), ... } — page found explicitly; rotation bootstraps if history empty
2. **Slug-matched new-path:** ApplyAction::Reingest { existing_page_id: None, ... } — page not in file_state but slug matches existing DB page; load_existing_page_identity finds it; rotation bootstraps

## Repair decision

Add pre-flight guard in apply_reingest BEFORE any pages or raw_imports mutation:

- After load_existing_page_identity() resolves current_page
- If current_page is Some(existing_pid, _), query COUNT(*) FROM raw_imports WHERE page_id = existing_pid
- If row_count == 0, return ReconcileError::InvariantViolationError immediately

This is the narrowest repair: only fires for existing pages, runs pre-mutation so transaction undirtied, uses same InvariantViolationError variant.

**New pages (current_page = None → INSERT → created = true) unaffected.**

## Implementation

Function: apply_reingest in src/core/reconciler.rs
- Added call to preflight check after load_existing_page_identity
- Check: if page found, assert_existing_active_row_invariant called
- Error variant: InvariantViolationError("Existing page {} has zero total raw_imports history")
- Logic: returns early, no page/file_state/raw_imports mutation occurs

## Rationale

The assert_existing_active_row_invariant function in raw_imports.rs is correctly designed to allow row_count == 0 for genuine new-page creation. The fix must live in apply_reingest (the caller) where context about whether the page is new or existing is available, not in rotate_active_raw_import itself (which is also called from the ingest path for real new pages).
