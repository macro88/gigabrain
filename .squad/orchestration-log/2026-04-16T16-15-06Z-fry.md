# Orchestration Log — Fry (Main Engineer, Model Release Review)

## Implementation Feasibility Assessment

**Verdict:** Feasible for two BGE-small variants only; reject current `--model small|base|large` UX.

## Technical Breakdown

### Two Variants (Approved Scope)

**Variant 1: Airgapped (Current)**
- Embed full BGE-small-en-v1.5 model weights (~90MB) via `include_bytes!()`
- Single static binary ~180MB (with debug symbols)
- Zero network required; safe for airgapped environments

**Variant 2: Online Slim**
- No embedded model weights
- Init flow: on first `gbrain query` or `gbrain embed`, download BGE-small from Hugging Face
- Binary size ~90MB (no weights)
- Network required; significantly faster installation

### Global `--model small|base|large` Rejection

**Why Not Viable Now:**
1. **Schema Shape Variance**
   - BGE-small: 384 dimensions → `vec_384` table + embedding metadata
   - BGE-base: 768 dimensions → `vec_768` table + embedding metadata
   - BGE-large: 1024 dimensions → `vec_1024` table + embedding metadata
   - Different column counts, different row shapes

2. **Re-embedding Cost**
   - User switches from small (384) to base (768): all 10K pages need re-vectorization
   - Current inference is hash-shim; production will be real candle forward-pass — expensive at scale
   - No efficient downgrade path (large → small would lose information)

3. **Release/Test Explosion**
   - CI matrix: 6 combinations (airgapped+{small,base,large}, online+{small,base,large})
   - Install paths: different pre-built artifacts, different download strategies
   - npm publish: multiple package variants or conditional postinstall logic
   - Bender's validation burden: 3x the rollout scenarios

## Recommended Build Changes

1. **Single Cargo feature:** `--features embedded-model` (default)
   - When enabled: current (weights embedded, 180MB binary, offline)
   - When disabled: slim binary (~90MB), init-time download

2. **No clap `--model` flag** in this release
   - Keep CLI surface simple
   - Init and config use hardcoded BGE-small model name
   - Re-embedding via export/import handled in Phase 4

3. **Binary naming:**
   - `gbrain-0.9.0-x86_64-unknown-linux-musl` (airgapped, embedded weights)
   - `gbrain-0.9.0-slim-x86_64-unknown-linux-musl` (online, no weights)
   - npm packages: single `gbrain` package with smart postinstall detection

## Impact on Release Workflow

- `release.yml`: Two build jobs (one per feature set)
- GitHub Releases: Two downloadable artifacts per platform
- npm publish: Single package, conditional init behavior
- docs/install.md: Branch on platform capability vs. network preference
