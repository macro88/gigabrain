### 2. **Core vault sync** (`src/core/vault_sync.rs`)
- Plain sync path wired as active-root stat-diff + reconciler entrypoint

- Collection liveness checked at entry; lease verified throughout
- Short-lived owner lease released on panic and error paths (via RAII guard)
- Explicit fail-closed gates for restore/remap/finalize pending states

### 3. **Reconciler** (`src/core/reconciler.rs`)
- `DuplicateUuidError` halt stops mutation; operator must `reconcile-reset`
- `UnresolvableTrivialContentError` halt stops mutation; operator must `reconcile-reset`
- Deduplication and content resolution proceed only on non-halted collections
- All halts are terminal; no self-healing or best-effort partial clears

### 4. **CLI truth surface** (`gbrain collection info`)
- `blocked_state` field distinguishes restore-pending / restore-integrity / reconcile-halted / manifest-incomplete
- `integrity_blocked` boolean for operator quick-check
- `suggested_command` points to correct next action (`restore-reset`, `reconcile-reset`, `finalize-pending`)
- JSON output exposes all truth fields for downstream agent consumption
- No MCP `brain_collections` surface added (deferred; CLI-only as approved)
